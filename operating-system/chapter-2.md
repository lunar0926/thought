# chapter 2 운영체제 개요

## 운영체제의 역할

* 크게 4가지가 있다.
* 뒤에 운영체제의 기능이 나오는데 둘을 구분해서 이해하기. 역할은 운영체제가 맡은 임무? 에 해당하고 기능은 운영체제가 구체적으로 하는 일을 의미한다.
* User Interface\(편의성\)
* Resource management\(효율성\)
* Process and Thread management
* System management\(시스템 보호\)
* 운영체제에 대해 이해하기에 앞서 대략적인 컴퓨터 시스템을 파악하고 있어야 한다. \(큰 그림\)

## 운영체제의 구분

동시 사용자 수 / 동시 실행 프로세스 수 / 작업 수행 방식 에 따라서

여기서 작업 수행 방식에 따른 운영체제 구분은 운영체제의 발전 역사를 통해서 이해할 수 있다.

* batch processing system, time-sharing system, distributed processing system, real-time system 등이 있는데 각각은 이전 작업 수행 방식을 보완하고자 나타났음을 이해하기.
* 장, 단점 이해.

## 운영체제의 구조

* 커널; 알맹이라는 뜻으로 운영체제의 핵심 부분. **메모리 상주**
* 유틸리티: **비상주** 프로그램.
* 운영체제의 역할은 자원을 관리하는 것임을 상기하기!

커널의 주요 기능 중 자원 관리가 있다. 하드웨어의 자원과 소프트웨어의 자원을 모두 관리한다.

운영체제의 구조는 단일 구조와 계층 구조, 마이크로 커널 구조가 있다.

* 각각의 특징과 장, 단점 이해

## 운영체제의 기능

* 이거는 외워두기\(6가지\)
* 프로세스 관리
* 프로세서 관리
* 메모리 관리
* 파일 관리
* 입출력 관리
* 보조 기억 장치 및 기타 주변장치 관리 등
* 각 기능에 대해서는 이후 내용에서 자세히 다루기 때문에 대략적으로만 이해\(핵심 키워드라고 할 수 있음!\)
* ex\) 메모리 관리에서 메모리 할당 방법에는 전체 적재와 일부 적재가 있는데 여기서 일부 적재는 프로그램 및 데이터의 일부만 메인 메모리에 적재하고 나머지는 보조기억 장치에 접근하는 가상메모리 개념.

others...

* System call interface는 자주 나오는 용어이므로 기억하고 있기
* 응용 프로그램과 OS 사이의 인터페이스로 OS가 응용프로그램에 제공하는 서비스이다.
