# 트리\(구현, 시각적인 설명 추가하기\)

### 트리 자료구조

 트리 자료구조는 노드와 노드의 연결로 표현한다. 그래픽 자료구조의 일종으로 **데이터베이스 시스템**이나 **파일 시스템**과 같은 곳에서 많은 양의 데이터를 관리하기 위한 목적으로 사용한다. 

![&#xD2B8;&#xB9AC; &#xAD6C;&#xC870;&#xC758; &#xD2B9;&#xC9D5;](../.gitbook/assets/image%20%281%29.png)

* 트리는 부모와 자식 노드로 구성.
* 최상단 노드를 루트 노드라고 함.
* 최하단 노드는 단말 노드.
* 트리에서 일부를 떼어내도 트리 구조를 이루며, 이를 서브 트리라고 함.
* 트리는 파일 시스템과 같이 계층적이고 정렬된 데이터를 다루기에 적합하다.

### 이진 탐색과의 연관성

 이진 탐색은 데이터가 정렬되어 있을 때 빠르게 탐색할 수 있는 알고리즘이다. 또한 데이터의 입력이 많고, 탐색 범위가 큰 상황에서 이진 탐색이 효과적이다. 

 데이터베이스는 내부적으로 트리 자료구조를 이용한다. 트리 자료구조는 항상 데이터가 정렬되어 있으며 이진 탐색과 유사한 방식으로 탐색을 빠르게 수행하도록 설계되어 있어서 데이터가 많아도 탐색하는 속도가 빠르다. 

### 이진 탐색 트리 

트리 자료구조 중에서 가장 간단한 형태가 이진 탐색 트리이다. 트리의 여러 종류 중 하나이다. 이진 탐색 트리는 이진 탐색이 동작할 수 있는, 효율적인 탐색이 가능한 자료구조이다.

 이진 탐색 트리에서 데이터 값은 왼쪽 자식 노드 &lt; 부모 노드 &lt; 오른쪽 자식 노드 순이다.

 어떤 값을 찾을 때 부모 노드를 기준으로 더 큰 지 작은 지를 비교하여 바로 왼쪽 또는 오른쪽 자식 노드를 방문하여 탐색할 수 있다. 이 과정을 반복하여 값을 찾으면 탐색을 마친다. 자식 노드가 없을 때까지 탐색해도 찾지 못하였다면 해당 값을 가진 원소가 없는 것이다.
