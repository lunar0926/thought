# 파이썬 가상환경

## 파이썬 가상환경을 만드는 이유

 로컬 디렉토리 만들고 거기서 바로 프로젝트를 하면 될텐데 굳이 가상환경에서 하는 이유는?

* 프로젝트를 만들 당시의 패키지 버전이 있는데, 이후에 패키지가 업데이트 된다면 프로젝트의 패키지와 호환이 되지 않을 수 있음. 그렇다고 로컬에서 패키지를 업데이트 하지 않고 그대로 놔둘수도 없음.
* 프로젝트마다 패키지의 버전이 다를 수 있어서 패키지를 프로젝트 별로 구분 해야 함.
* 파이썬의 버전이 달라질 수도 있음. 프로젝트를 만들 당시의 파이썬은 업데이트된 새로운 버전의 파이썬으로 실행했을 때 호환이 되지 않거나 제대로 결과가 나오지 않을 수 있음. 따라서 가상환경에서 파이썬 버전을 명시해두고 가상환경 내에서 패키지, 모듈을 설치하는 것.



## 가상환경 만들기

프로젝트를 진행하는 디렉토리로 이동한 다음에\(또는 디렉토리를 만들어서 이동한 다음에\)

디렉토리를 만드는 명령어는 `mkdir 디렉토리이름`

&lt;가상환경 설치&gt;

맥

`virtualenv -p python3 venv`

 윈도우

`virtualenv venv`

여기서 venv는 임의로 내가 설정한 가상환경의 이름. 다른 걸로 해도 상관없다. 

## 가상환경 진입

맥

`source venv/bin/activate`

윈도우

`venv\Scripts\activate`

성공하면 \(venv\) 와 같이 가상환경이 표시된다.  

이 가상환경 안에서 pip install django 로 모듈 설치

## 프로젝트 시작

해당 디렉토리의 가상환경에서

django-admin startproject 프로젝트이름 . \(.도 포함해야함. 현재 디렉토리라는 의미\)

python [manage.py](http://manage.py) runserver \(실행\)
